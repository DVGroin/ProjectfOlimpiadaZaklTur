### 💡 Советы для максимального количества баллов

1.  **История коммитов — это доказательство работы.**
    Не делайте один коммит «Всё готово» в конце. Жюри смотрит `git log`. Делайте коммиты каждые 20-30 минут с понятными сообщениями:
    ```bash
    git add .
    git commit -m "feat: добавлена валидация входных данных"
    git commit -m "fix: исправлена ошибка подключения к БД"
    git commit -m "refactor: вынесена логика хеширования в отдельный модуль"
    ```

2.  **Проверка импортов (страховка от сбоев).**
    Создайте файл `tests/check_imports.py`, чтобы за 1 минуту проверить, что все библиотеки встали корректно:
    ```python
    # tests/check_imports.py
    import sys
    libs = ['tensorflow', 'keras', 'flask', 'sqlalchemy', 'PyQt5', 'numpy', 'pandas']
    failed = []
    for lib in libs:
        try:
            __import__(lib)
            print(f"✓ {lib}")
        except ImportError:
            print(f"✗ {lib} — НЕ НАЙДЕН")
            failed.append(lib)
    sys.exit(1 if failed else 0)
    ```
    Запуск: `python tests/check_imports.py`. Если всё ок — выход 0, если есть ошибки — выход 1.

3.  **Чистота кода.**
    В `main.py` оставляйте только точку входа. Основную логику выносите в папку `src/` или отдельные файлы. Это покажет жюри, что вы умеете структурировать проект.

Нужен пример структуры для конкретного типа задачи (веб-сервис, десктоп или анализ данных)? Напишите, какой кейс вы планируете решать.
